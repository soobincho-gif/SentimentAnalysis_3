# UI Package

- Purpose: Explain when shared UI assets deserve promotion out of the web app.
- Owner: Frontend maintainers.
- When to read: Before extracting shared design tokens or reusable UI components.
- Decisions that belong here: Shared UI scope and the threshold for introducing reusable UI packages.

## Scope

`packages/ui` is optional for the MVP. Use it only when the web app develops shared tokens, components, or UI utilities that justify reuse outside a single local screen implementation.
