# 루프 운영 메모

## 해석

사용자가 원하는 "자동화"는 우선적으로 반복 가능한 작업 루프와 컨텍스트 유지 구조로 해석했다.

즉, 이번 단계에서는:

- 실행 루프 프롬프트를 만들고
- 상태 보드와 컨텍스트 브리프를 추가하고
- 이후 작업마다 같은 형식으로 로그를 누적할 수 있게 만들었다.

## 아직 하지 않은 것

- 특정 시간에 자동으로 실행되는 스케줄형 자동화 생성

## 이유

- 반복 실행 시점과 주기를 아직 정하지 않았기 때문이다.
- 현재는 먼저 "어떻게 반복할지"를 문서와 프롬프트로 고정하는 것이 더 중요하다.

## 나중에 추가 가능한 자동화 예시

- 매일 오전에 현재 로그를 읽고 다음 구현 루프 제안
- 매 작업 종료 후 변경 로그와 리스크 요약 생성
- 주간 단위로 planning 대비 실제 진척 요약 생성
