# 해결한 항목

## 1. 디자인 문서 형식 불명확성 해결

- 기존 `DESIGN.md`는 방향성은 좋았지만 `awesome-design-md`가 강조하는 9개 섹션 구조와 에이전트 프롬프트 가이드까지는 반영되지 않았다.
- 이를 보완해 시각 분위기, 색상 역할, 타이포, 컴포넌트, 레이아웃, 깊이, 가이드라인, 반응형, 에이전트 프롬프트 가이드로 재구성했다.

## 2. 보고용 기록 부재 해결

- 기존에는 작업 결과는 있었지만 보고서용으로 재활용하기 좋은 로그 폴더가 없었다.
- `logs/` 규칙과 날짜별 작업 폴더를 추가해 계획, 해결 항목, 변경 로그, 리스크, 참고 소스를 분리 기록할 수 있게 만들었다.

## 3. 환경 변수 보안 기본값 해결

- 새 API 키를 반영해야 하지만 실수로 공유 파일에 남을 위험이 있었다.
- 실제 키는 `.env`에만 넣고, `.gitignore`로 추적 방지 설정을 추가했다.

## 4. 저장소 문서 경계 명확화 보강

- `AGENTS.md`에 작업 로그 규칙을 추가해 앞으로도 구조적 작업에는 기록이 남도록 했다.
- `README.md`에도 `logs/` 폴더 목적을 명시했다.
