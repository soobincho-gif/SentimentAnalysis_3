# 작업 계획

## 목적

- 저장소 문서 계층을 먼저 고정한다.
- `awesome-design-md`의 구조를 반영해 `DESIGN.md`를 실전형 디자인 시스템 문서로 강화한다.
- 이후 보고서 작성에 쓸 수 있도록 작업 로그를 한 폴더에 정리한다.
- 새 OpenAI API 키를 로컬 환경 파일에 반영한다.

## 이번 작업 범위

1. 외부 레포 구조 확인
2. `DESIGN.md` 재구성
3. 로그 체계 추가
4. `.env` 및 `.gitignore` 정리

## 작업 원칙

- 디자인 문서는 룩앤필 전용으로 유지한다.
- 개발 규칙과 디자인 규칙을 섞지 않는다.
- 로그는 보고 자료로 재사용 가능하도록 요약 중심으로 남긴다.
- 비밀 값은 실제 `.env`에만 두고 로그에는 남기지 않는다.
