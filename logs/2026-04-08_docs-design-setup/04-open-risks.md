# 남은 리스크와 확인 필요 사항

## 리스크

- 아직 실제 앱 UI 코드가 없어서 `DESIGN.md`가 구현 코드 수준에서 검증되지는 않았다.
- 모델 선택값이 아직 비어 있어 실제 API 구현 시 분석용 모델과 생성용 모델을 정해야 한다.
- 현재 저장소는 Git 초기화 상태가 아니므로 비밀값 추적 위험은 낮지만, 이후 Git 사용 시 `.env` 관리 규칙을 유지해야 한다.

## 확인 필요 사항

- 이미지 분석용 모델과 스토리 생성용 모델을 무엇으로 고정할지
- 첫 UI에서 이미지 재정렬을 지원할지 여부
- 기본 스토리 길이와 재생성 UX의 첫 범위를 어디까지 둘지

## 다음 권장 작업

2. `packages/prompts`에 분석/생성 프롬프트 계약 추가
3. `packages/services`에 서비스 인터페이스 뼈대 추가
