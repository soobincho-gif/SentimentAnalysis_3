# Tests

- Purpose: Explain how repository-wide tests should be organized.
- Owner: Engineering team.
- When to read: Before adding new tests or fixtures.
- Decisions that belong here: Test folder purpose, broad grouping, and fixture conventions.

## Scope

This folder will contain unit, integration, contract, and future UI flow tests that validate behavior across packages and apps.

## Rule

Prefer test folders named after the level of behavior being verified, not after vague ownership buckets.
